## Build provenance

`build/bin/gelu_mgpu_align` rebuilt on MN5 immediately before submission.
The library dependencies relinked were:

- `libphantom_ext.a` — unchanged this iteration.
- `libnexus_multi_gpu.a` — relinked against the post-FIX-BUG-03-01 source
  (`src/multi_gpu/comm/nccl_comm.cu` and
  `src/multi_gpu/distributed_context.cu`, commit `05445b6`); the two
  cleanup-order risks in `DistributedContext::destroy()` are now fixed,
  so any teardown-time GPU error during multi-rank gelu finishing is
  surfaced cleanly instead of crashing the trace.
- `libnexus_evaluators.a` — relinked against the post-FIX-BUG-04-04 source.
  GELU itself does not use `multiply_power_of_x`, but the link still
  resolves into the new lib so the binary is consistent with the rest of
  Phase C's submissions.

Build was incremental (`make -j20 gelu_mgpu_align`) on `cuda/12.8 +
cmake/3.30.5 + nccl/2.24.3-1` and clean modulo pre-existing NTL header
warnings.
