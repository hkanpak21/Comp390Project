`build/bin/argmax_align_n32k` rebuilt on MN5 immediately before submission.
The library dependencies relinked were:

- `libphantom_ext.a` — unchanged this iteration.
- `libnexus_multi_gpu.a` — relinked against the post-FIX-BUG-03-01 source
  (`src/multi_gpu/comm/nccl_comm.cu` and
  `src/multi_gpu/distributed_context.cu`, commit `05445b6`); the two
  cleanup-order risks in `DistributedContext::destroy()` are now fixed,
  so any teardown-time GPU error during multi-rank argmax finishing is

  gate (decoded argmax must match plain argmax) are now enabled.

## Vocabulary constraint note

`argmax_align_n32k` at logN=15 is limited to single-cipher invocations:
`vocab ≤ sparse_slots = 8192`. This run uses `--vocab 8` as a safe
single-cipher baseline. The NEXUS published result (2.48 s) uses `vocab=30522`
(multi-cipher tournament), which is out of scope for this paper. Limitation
is disclosed in §8 Discussion.
